
# ASIS vs TOBE 마이그레이션 비교 프로젝트

프로젝트 목표: 100개 테스트 케이스에 대해 ASIS 기존 시스템 출력과 TOBE 신규 시스템 출력을 자동으로 비교하여 마이그레이션 정확성 검증

1. 프로젝트 개요

1.1 배경
기존 메인프레임 시스템(ASIS)에서 신규 PITON 기반 (확장 미들웨어) (TOBE)으로 마이그레이션 진행 중이고 마이그렝이션이 잘 되었는지 확인하는 작업을 실시해야 한다.
ASIS는 고객이 테스트를 실시하고 동일한 테스트 데이터를 이용해서 TOBE환경에서 테스트를 실시후 동일한지 확인하는 작업을 해야한다.
TOBE작업을 전부 자동화 해서 테스트 시간을 단축하는 것을 목표로 한다.

1.2 전제조건
TOBE환경에서 테스트를 실시하고 결과를 비교하기 위한 이하의 정보가 TOBE의 현신비교용 ASIS디렉토리에 저장되어 있는 것을 기본으로 한다.
- ASIS의 테스트 체크리스트
- ASIS의 각 체크리스트를 테스트하기 휘한 입력데이터(DB, 파일)를 파일로 다운로드 한후에 문자코드 변환 정보(ABCDIC로부터 Shift_JIS로 변환)
- ASIS의 각 체크리스트를 테스트 한 후의 출력데이터(DB, 파일)를 파일로 다운로드 한후에 문자코드 변환 정보(ABCDIC로부터 Shift_JIS로 변환)

1.3 목표
이하의 처리를 하는 프로그램을 작성한다.
정의 파일에는 복수건의 테스트를 반복해서 처리해야 할 기본정보가 등록되어있다. 예를 들면 입력데이터가 저장된 디렉토리, 입력용 테이블 및 파일명, 해당테이블을 등록하기 위한 DB커넥션 관련 정보, 실행해야 할 Shell프로그램 명, Shell프로그램에 등록된 코볼 프로그램이 출력한 출력 테이블 및 파일명등이 정의파일에 저장되어 있다.
정의파일을 읽어서 아래의 자동화 처리 프로그램을 작성해서 복수건의 테스트 처리를 반복해서 자동화 처리를 실현한다.

- 정의 파일에서 필요정보 취득
- DB커넥션
- 테스트용 입력데이터가 DB인경우 해당 데이블에 UPLOAD, 파일인 경우는 TOBE환경의 해당 디렉토리에 복사 
- SHELL프로그램 기동

1.4 범위
- 테스트 케이스: 100건
- 처리 방식: Shell 스크립트로 자동화 배치 처리
- 데이터 저장소: PostgreSQL

2. 환경
2,1 ASIS 시스템 환경
- 기반 플랫폼: 富士通メインフレーム（OS：MSP）
- 프로그래밍 언어: COBOL85 （BAGLESⅡより自動生成）
- DBMS: AIM/NDB （ネットワークDB）
- DC: AIM/DC

2.2 TOBE 시스템 환경
- 기반 플랫폼: PITON 기반 (확장 미들웨어) 
- 프로그래밍 언어: Net COBOL
- DBMS: PostgreSQL (RDB 변환)
- 배치 실행: 배치 Shell → 배치 프로그램 → 처리 결과 DB 저장
