# DESIGN_TOKENS — 現新比較ツール GUI 디자인 토큰 (정본)

> ⚠️ **값 폐기 — 재작성 예정 (D-043 / 보정).** 본 제품은 ModernizePro Compare로 형제 제품 ModernizePro의
> **셸 구조(사이드바+탭)는 공유**하되, **주색은 녹색이 아닌 Compare 전용 색(색상명 미정)으로 분기**한다.
> 아래 먹네이비(#1F2937) 등 자체 토큰은 **무효**이며, 형제 제품 스크린샷 기반으로 재작성 예정(정식 가이드 미확보).
> 착수=실배치 검증 후 + 스크린샷 확보 후. 그때까지 참조 금지.

> UI는 글로 핸드오프하면 어긋난다. 이 문서가 **값의 정본**이다. Claude Code는 추측·해석 없이
> 아래 값을 CSS 변수로 그대로 옮긴다. "버튼 모양이 다르다"가 재발하면 이 문서 기준으로 맞춘다.
> 방향(확정): 먹색/네이비 주색 · 중간 질감(적당히 둥글고 밀집) · 상태색 절제 · 단계상태는 텍스트로.

## 색 (hex 정본)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--c-primary` | `#1F2937` | 주색(먹네이비). **주요 액션 버튼 채움**(検証実行) |
| `--c-primary-fg` | `#FFFFFF` | 주색 위 글자 |
| `--c-accent` | `#2F4A6B` | 보조버튼 글자·링크·강조 텍스트 |
| `--c-text` | `#1F2937` | 본문 강(제목·수치) |
| `--c-text-mut` | `#6B7280` | 본문 약(설명·라벨) |
| `--c-text-faint` | `#9AA1AB` | 힌트·비활성 |
| `--c-surface` | `#FFFFFF` | 카드 면 |
| `--c-surface-2` | `#F4F5F7` | 보조 면(요약줄·수치카드) |
| `--c-border` | `#E2E5EA` | 기본 테두리(연) |
| `--c-border-strong` | `#CFD5DD` | 활성 카드 테두리(진) |
| **상태색(절제 — 형광 금지)** | | |
| `--c-ok` | `#3B7A57` | OK·점검통과(차분한 초록) |
| `--c-ng` | `#A83232` | NG(가라앉은 적) |
| `--c-error` | `#B06A1B` | ERROR(가라앉은 호박) |

상태색은 **수치/배지/한 줄 텍스트에만** 점적으로. 면을 채우지 않는다(경고가 비명 지르지 않게).

## 형태 (중간 질감)
- 카드: `border-radius:10px`, `padding:14px 16px`, `border:0.5px solid var(--c-border)`.
  지금 진행 중인 단계 카드만 `--c-border-strong`. 완료·대기 단계는 `--c-border`.
- 버튼 공통: `border-radius:6px`, `border:0.5px solid`, `font-weight:500`.
  - 주요(検証実行): bg `--c-primary` / 글자 흰색 / `padding:10px 20px` / `font-size:14px`.
  - 보조(事前点検·変更·Excel): bg 흰색 / 테두리 `--c-border-strong` / 글자 `--c-accent` / `padding:8px 14px` / `13px`.
- 수치 카드(件数): bg `--c-surface-2`, `border-radius:6px`, `padding:10px 12px`, 라벨 11px `--c-text-mut`, 수치 21px/500.
- 요약줄(접힌 단계·접속설정): bg `--c-surface-2`, `border-radius:8px`, `padding:10px 14px`, 한 줄.

## 스텝 숫자 뱃지 (결정: B — 단색 통일)
- **전 단계 동일**: `width/height:26px`, 원형, bg `#E8EDF3`, 숫자색 `--c-accent`, `12px/500`.
- 숫자는 **순서 표시 전용.** 단계 상태(완료/현재/대기)는 뱃지 색이 아니라
  **접힘+텍스트**로 표현한다(완료=한 줄 요약 접힘, 현재=펼침, 상태문구 "点検OK·エラー0/警告1").
- 단계 간 연결선: `0.5px` 세로선, `--c-border`.

## 상태 표현 (결정: 단계별 최소 텍스트)
- 색·아이콘으로 도배하지 않는다. 한 줄 텍스트가 주, 아이콘은 보조.
  예: `✓ 点検OK · エラー0 / 警告1`(--c-ok), `完了 · sample_real.csv（CK001）`.
- 件数만 상태색 적용(OK/NG/ERROR 수치). 나머지는 --c-text 계열.

## 타이포·아이콘
- 폰트: 시스템 산세리프(일본어 포함). 굵기 **400/500 두 단계만**(700 금지 — 무거움).
- 제목 15px/500, 본문 13~14px/400, 라벨 11~12px.
- 아이콘: Tabler outline. 인라인 14~17px. 색은 부모 상속(상태 아이콘만 상태색).

## 적용 규칙
- 이 값들을 CSS 변수(`:root`)로 정의하고 전 화면이 변수만 참조. 하드코딩 hex 산재 금지.
- **화면(템플릿/뷰/CSS)만 변경. 라우트·로직·스키마 손대지 말 것.** 255 테스트 녹색 유지.
- 시안(채팅 렌더)은 폰트·미세여백이 실제와 다를 수 있음 — **이 문서의 값이 정본.**
