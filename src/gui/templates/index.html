Project Settings 接続・環境設定（環境ごと1回）

接続設定（DB）

検証専用 DB／最小権限アカウントで実行してください（入力テーブルを TRUNCATE します）。パスワードは保存せず環境変数から読みます。

ディレクトリ設定（paths）

As-Is / To-Be データの場所をここで指定します（環境ごと1回）。空欄は既存値を保持。相対パスは設定ファイル基準。 項目ごとに場所が違う場合のみ、定義の *_dir で行単位 override。保存後の存在確認は 事前点検。

業務別ディレクトリ（任意）

業務ごとに場所が違う場合
業務（shell_group）ごとに As-Is/To-Be ディレクトリが違う場合に指定します。経路解決: 項目 override（定義 *_dir）＞ 業務ディレクトリ ＞ 全体 paths。空欄は全体 paths にフォールバック。
